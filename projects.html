AZ Platform Project Detail

It is a small social media platform where users can read the posts uploaded by admins. Both user and admin are separated by roles and they have their different authorizations. Admin side includes creating posts, retrieving posts, retrieving user, admin informations and profies, comments, reacts and admin to admin conversations. Users can only view posts, make comments and reacts. Below is the most important key structure, database ERD design of the system.

Admin view

At first, there will be an admin existing inside database. A person can become an admin by first registering on the admin register page. And then, it will be a pending stage for that person because to become an admin, it needs one of the existing admins' approval. Then, he can login to the web page with this own user name and password. All of the passwords are hashed so that even if the database is hecked, password cannot be revealed. Admin view is a server-side-rendered website, therefore server will store information about logged in user in its database and user has to store it in browser's cookie. Remember me button can be used to generate a long-lasting cookies.

This page is just like facebook, I made nav bar in the left side and the right side is about users and admins, where admin can chat with other admins live.

Admin can create, modify and delete categories in category page.

Admin can create, modify and delete posts in post page. Not like newsfeed home page, it will display posts with table in overview format.

Admin can view post details of the post including who comment and react posts.

The list of all users is displayed with tabular format and admin can ban users in certain period. If a user's banning period is over, it will show the green sign.

The list of admin can also be seen. Can clicking 'i' button, you can go to admin's profile.

In each profile, the list of posts that admin has posted are shown.

Chat page is one of my favorite and the one I am proud of the most. Chat page is almost the same with messenger app from facebook. It uses javascript ajax to call functions in every second expecting that the other person might send a new message. A person can remove his message by clicking trash bin button.

User view

From the side of a user, he can only view, give reactions and comment the post. He is not allowed to create posts. Users can create account and login if already exists. This website is a token based website which is separate from backend files and only have connections through APIs. Once the user has logined successfully, he will get a token from backend and will continue other APIs using that token.

Post list is the place user can see the posts and finding, searching with categories. By clicking the view, post detail page will be seen.

New Feeds is a page where users will see posts one by one just like facebook's newsfeed. There, user can comment, react to the post or can view the post for more detail.

Post detail page is where user can see the detail description of the post and comments and reacts. User can give comments or delete or edit his comment if he wants.
